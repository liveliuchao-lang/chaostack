8️⃣ 已知限制、风险与解决方案

8.1 Flash Attention 非原生

Flash Attention 通过 Triton 桥接，非原生 ROCm 内核。

🛠️

  • vLLM Triton Attention 后端（VLLM_V1_USE_PREFILL_DECODE_ATTENTION=1）
  • 回退 AITER MHA 内核（VLLM_ROCM_USE_AITER_MHA=1）
  • 上下文控制在 32K 以内

8.2 HuggingFace TGI/TEI 不支持

纯 CUDA 独占，无 AMD 构建。

🛠️

  • 推理：替换为 vLLM
  • 嵌入：PyTorch + AutoModel（ROCm 下 device='cuda' 自动映射）
  • 重排序：FlagEmbedding 库

8.3 TensorRT-LLM 不可用

🛠️

  • MLC-LLM（TVM 编译优化，功能最接近）
  • vLLM V1 引擎 + full CUDA graph 模式
  • 期望性能：NVIDIA TensorRT-LLM 的 60-80%

8.4 xFormers 支持滞后

🛠️

  • vLLM flash_attn 和 MLC-LLM 已覆盖大部分用例
  • ComfyUI: --force-fp16 --disable-xformers 使用 PyTorch SDPA
  • ROCm fork: pip install git+https://github.com/ROCm/xformers.git

8.5 DeepSpeed/FSDP 多卡训练不成熟

🛠️

  • W7900D 是推理卡，不推荐多卡训练
  • 单卡 QLoRA 微调（48GB 可微调 70B Q4）
  • 小模型（7-13B）单卡全参数微调

8.6 llama.cpp HIP 慢于 Vulkan

🛠️

  • 方案A：使用 Vulkan 后端（-DGGML_VULKAN=ON）
  • 方案B：rocWMMA 补丁（Green-Sky fork）
  • 方案C：直接用 MLC-LLM（快 35%）

8.7 70B MLC-LLM 编译 40 分钟

🛠️

  • 首次后缓存，后续加载秒级
  • 频繁换模型用 vLLM（免编译）
  • Docker 镜像提前编译好

8.8 Windows 支持有限

🛠️

  • 推荐 Ubuntu 24.04.4 LTS
  • Windows 用户用 WSL2 + Ubuntu 24.04
  • HIP SDK 7.2 可满足 Ollama/ComfyUI 基础需求

8.9 RDNA3 矩阵核心效率低于 Tensor Core

🛠️

  • Q4/INT8 量化时差距缩小到 10-20%
  • 优先使用 FP8 精度（ROCm 7.0+）
  • 使用 vLLM AITER 融合内核

8.10 无 NVLink，多卡通信受限

🛠️

  • 双卡 tensor parallelism 可行（PCIe 4.0 ×16 够用）
  • 避免 4+ 卡；超过 2 卡建议 Instinct 系列
  • 纯推理场景通信量小，PCIe 4.0 足够
